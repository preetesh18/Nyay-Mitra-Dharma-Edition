## 🔍 Technical Specifications

### Knowledge Base
- **Total Passages**: 1,088
- **Bhagavad Gita**: 700+ verses with Chapter/Verse
- **Chanakya Niti**: 150+ entries
- **Vidura Niti**: 150+ entries
- **Hitopadesha**: 88+ stories
- **Encoding**: UTF-8 with Devanagari support

### AI Integration
- **Model**: Google Gemini 2.5-flash
- **Fallback Chain**: 2.0-flash → 2.0-flash-lite
- **Architecture**: Retrieval-Augmented Generation (RAG)
- **Retrieval Method**: TF-IDF with cosine similarity
- **Devanagari Boost**: 2x weight for Sanskrit matches

### Response Structure
**All responses include 7 sections:**
1. Direct Answer (to user's question)
2. Scripture Reference (exact passage)
3. Sanskrit Text (Devanagari + transliteration)
4. Explained Teaching (what it means)
5. Dharmic Guidance (3-5 action steps)
6. Modern Context (how to apply today)
7. Follow-up Prompt (encourage exploration)

### Performance Metrics
- **Model Init**: <2 seconds
- **Data Load**: <5 seconds
- **First Response**: <3 seconds
- **Subsequent Responses**: <2 seconds
- **Memory Usage**: ~500MB (with all data)
- **Scaling**: Multi-worker with Gunicorn

## 🎓 Learning Resources

### How It Works
1. User asks question (text or voice)
2. System retrieves 6 most relevant passages
3. Passages formatted per rules (Gita vs Others)
4. Gemini AI generates structured response
5. Response shown with 7 sections
6. Session saved for continuity

### Data Flow Diagram
```
User Query
    ↓
TF-IDF Retrieval (top 6 passages)
    ↓
Format Passages (Gita vs Others)
    ↓
Augment Prompt (add formatting rules)
    ↓
Call Gemini API
    ↓
Generate Response (7 sections)
    ↓
Show to User
    ↓
Save to Session Log
```
